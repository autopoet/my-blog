## 1. 组件通信方式

> 考察点：父子通信、跨层通信、全局通信
> 判断点：能说清每种方式的适用场景与原理

### 话术
> 组件通信根据层级和用途选择方案，父子通信用 props/emit，跨层用 provide/inject，跨模块用 Pinia，全局状态统一管理可提高可维护性。

### 1.1 父子通信

- 父传子：通过 props 向子组件传值
  - 特点：单向绑定，子组件不能直接修改父组件状态
  - 注意：子修改父数据需通过事件或回调
- 子传父：通过 emit 触发事件
  - 特点：事件驱动，保持组件解耦
  - 场景：表单组件向父传入用户输入

### 1.2 跨层通信

- provide / inject（组合式 API）
  - 祖先组件向其所有子孙组件“注入”数据，无论嵌套多深。
  - 后代组件向其祖先组件“请求”数据，无论嵌套多深。
  - 注意：仅适合静态或初始化状态，响应式需手动 wrap 成 reactive/ref

### 1.3 全局通信

- Pinia / Vuex
  - 管理全局共享状态
  - 跨模块、跨页面通信方便
  - 场景：登录用户信息、购物车、全局配置等

## 2. 动态组件与异步组件

> 考察点：提高组件复用性与性能
> 判断点：能说 <component:is>defineAsyncComponent、懒加载

### 话术
> 动态组件通过 <component :is> 可以在同一位置渲染不同组件，异步组件可按需加载，减少初始包体积，提高性能。

### 2.1 动态组件

场景：tab 页、步骤组件、条件渲染不同表单
示例：
~~~js
<component :is="currentComponent"></component>
~~~


### 2.2 异步组件

使用 defineAsyncComponent 或异步 import
优点：按需加载，减少首屏包体积
示例：
~~~js
import { defineAsyncComponent } from 'vue'
const AsyncComp = defineAsyncComponent(() => import('./MyComp.vue'))
~~~
场景：大型页面、弹窗组件、路由页面懒加载

## 3. 插槽（Slots）的应用场景

> 考察点：组件内容分发与复用
> 判断点：默认插槽、具名插槽、作用域插槽

### 话术
> 插槽用于内容分发，默认插槽简单替换，具名插槽适合多区域布局，作用域插槽可让父组件渲染子组件数据，实现高复用组件。

### 3.1 默认插槽

- 组件内部使用 <slot></slot>
- 场景：内容可替换，简单分发，通用容器/卡片

### 3.2 具名插槽

- 组件内部 <slot name="header"></slot>
- 场景：复杂布局，多区域可替换，导航栏，标准页面/弹窗布局： 比如一个 Modal（弹窗）组件，通常分为：头部（Header）、主体（Body）、底部（Footer）。

### 3.3 作用域插槽

- 痛点： 有时候，数据在子组件里，但样式（怎么画）得听父组件的。

- 例子： 封装一个列表组件 MyList。

  - 子组件负责：调接口、循环 v-for、处理分页。

  - 父组件负责：决定每一行是显示图片还是显示文字。

- 实现： 子组件通过 <slot :data="item"> 把数据“回传”给父组件。父组件通过 #item="slotProps" 接收数据。
