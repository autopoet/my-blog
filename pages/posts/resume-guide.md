对于前端校招/实习生来说，一个“能跑起来”的博客很普通，但一个“注重工程化实践”的博客很吸睛。你可以参考以下方式将该项目写在简历中。

## 1. 项目概况 (Project Summary)

**项目名称**：基于 SSG 架构的前端全栈技术博客
**核心架构**：Vue 3 + TypeScript + Valaxy + Decap CMS
**项目地位**：独立完成从 UI 深度定制、CI/CD 自动化部署到 Headless CMS 全栈集成的全流程开发。

---

## 2. 简历描述示例 (点对点)

你可以在直接根据你的面试方向选择以下描述词：

### 💡 关键词：全栈思维 & 认证授权

> “针对静态站点（SSG）无法处理 OAuth 授权的痛点，使用 **Node.js** 开发了 **Serverless Functions** 认证桥接器，通过处理 GitHub 授权令牌交换流程，实现了全栈闭环的 Web 端内容管理。”

### 🏗️ 关键词：工程化 & CI/CD

> “搭建了基于 GitHub Actions 与 Vercel 的 **CI/CD 自动化工作流**，实现了 Web 端内容发布到线上环境自动构建与 CDN 全球分发的无缝衔接。”

### ⚡ 关键词：性能与渲染

> “采用 **SSG (静态站点生成)** 渲染技术，配合 **UnoCSS (原子化 CSS)** 进行样式拆分与按需加载，极大地优化了 Web 指标（LCP/FCP），提升了移动端访问体验。”

## 3. 面试官可能会问的问题 (Q&A)

提前准备好这些答案，会让你的项目显得更有“含金量”：

**Q：为什么选择 SSG 架构而不是直接用现成的知乎/语雀？**

* 答：SSG 架构（静态站点生成）拥有极致的加载性能和 SEO 友好，且内容存储于 GitHub 仓库，实现了真正的“内容即资产”。通过集成 Decap CMS，我既保留了本地 Git 版本的强大控制力，又获得了媲美 SaaS 平台的网页端编写体验。

**Q：你在集成 CMS 时遇到最大的困难是什么？是怎么解决的？**

* 答：最大的挑战在于 OAuth 授权。因为静态页面没有后端接收 GitHub 的回调 code。我通过在 Vercel 部署 **Serverless Functions** 作为认证中转桥接（Auth Bridge），在函数层完成 token 交换，并利用 `vercel.json` 解决了管理路径下的 SPA 路由屏蔽问题。
