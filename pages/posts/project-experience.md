### 项目名称：高性能全栈技术博客与可视化管理系统
**技术栈**：Vue 3 (Composition API) + Valaxy (SSG 引擎) + Decap CMS + GitHub OAuth + Serverless Functions + UnoCSS

**项目描述**：
该项目是一个基于 Vue 3 生态构建的高性能技术博客系统，旨在解决传统静态博客管理繁琐的痛点。通过深度打通 Headless CMS 链路并自建全栈鉴权中转流，实现了“代码级控制”与“生产级可视化创作”的完美结合。

**核心贡献与技术要点**：
1.  **高性能架构实现**：
    *   采用 **SSG (Static Site Generation)** 架构，利用 Vite 驱动的预渲染技术将内容在构建阶段转化为静态 HTML。
    *   配合 **UnoCSS** 的原子化 CSS 特性进行按需扫描，大幅减少样式体积，结合 LCP 优化策略，实现首屏毫秒级响应。

2.  **全栈 Auth 鉴权闭环**：
    *   针对纯静态站点无法安全处理 Secret 的痛点，自主开发了基于 Node.js 的 **Serverless Functions** 认证中转服务。
    *   通过自定义授权端点安全地打通了浏览器端与 GitHub API 的令牌交换链路，实现了 Web 端可视化登录与管理。

3.  **可视化内容建模与自动化流**：
    *   深度集成 **Decap CMS**，通过 YAML 配置对 Markdown Schema 进行精准建模。
    *   实现了图片自动上传托管（GitHub API 回写）与文章分类映射。配合 **GitHub Actions** 构建了从 Web 端起草保存到 Vercel 自动化部署的完整 CI/CD 流转。

4.  **工程化与组件化定制**：
    *   基于 Valaxy 插件系统深度定制主题布局，针对面试求职场景构建了结构化的知识点标签系统。
    *   利用 TypeScript 进行类型约束，确保了项目在扩展自定义组件时的健壮性。

### 面试核心 Q&A（准备建议）：
*   **Q：为什么要自己写 Serverless 认证中转，而不是直接用现成的？**
    *   **A**：因为 GitHub OAuth 需要 `client_secret`，放在前端会泄露。为了保持站点“全静态”的同时具备后台管理能力，我选择用轻量级的 Serverless 函数作为“桥接器”，在保障安全的前提下完成了鉴权闭环。这体现了我对 Web 安全和全栈协作流程的理解。
*   **Q：SSG 相比于传统的 SPA 博客有什么优势？**
    *   **A**：SEO 友好、访问速度极快。所有内容在用户访问前就已经是 HTML 模板，极大地提升了用户体验，尤其是在弱网环境下的首屏展示。
