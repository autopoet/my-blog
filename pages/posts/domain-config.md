> 当你的博客成功部署在 Vercel 或 GitHub Pages 后，下一步通常是购买一个专属域名。这不仅是程序员的浪漫，更是建立个人品牌的基石。

## 1. 域名购买建议

-   **服务商选择**：国内推荐阿里云、腾讯云、华为云；国外推荐 Cloudflare, Namecheap, Google Domains (Squarespace)。
-   **后缀选择**：`.com` 是硬通货，`.me` 适合个人主页，`.io` 或 `.dev` 适合开发者社区。
-   **域名起名**：尽量简短、好记，最好包含你的个人 ID。

---

## 2. DNS 解析基础知识

DNS 像是一个大账本，把域名对应到服务器 IP。

| 类型 | 记录值示例 | 作用 |
| :--- | :--- | :--- |
| **A** | `1.2.3.4` | 直接指向一个 IPv4 地址 |
| **CNAME** | `cname.vercel-dns.com` | 将一个域名指向另一个域名（别名） |
| **AAAA** | `2001:0db8::` | 直接指向一个 IPv6 地址 |
| **MX** | `mxbiz1.qq.com` | 设置你的专属域名邮箱 |

---

## 3. 在 Vercel 中连接域名

这是最简单的配置方式（以 Vercel 为例）：

1.  进入 Vercel 仪表盘 -> **Settings -> Domains**。
2.  输入你的域名（如 `blog.example.com`），点击 **Add**。
3.  Vercel 会显示两行配置信息：通常是一个 **A 记录** 或一个 **CNAME 记录**。
4.  复制这些值，回到你购买域名的管理后台。
5.  在 **DNS 解析** 页面，添加对应的记录。
6.  等待 5-30 分钟，即可完成解析。

---

## 4. HTTPS 安全证书 (SSL)

-   现在的浏览器强制要求 HTTPS 访问。如果使用 Vercel，它会**自动且免费**为你签发 Let's Encrypt 证书。
-   解析生效后，你的博客就会自动带上“小锁”。

---

## 5. 常见问题：WWW 与 根域名

-   通常我们会在 DNS 界面设置两个记录：
    -   `@` (根域名，如 `example.com`)
    -   `www` (如 `www.example.com`)
-   **最佳实践**：在 Vercel 中添加这两个域名，并将其中一个重定向到另一个，确保访问的一致性。

> [!TIP]
> 推荐使用 **Cloudflare** 作为 DNS 管理器。它不仅解析快，还提供免费的 Web 防火墙和无限解析记录。
